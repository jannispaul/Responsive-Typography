Responsive Typography

This site aims to collect the most relevant practices around responsive typography on the web and serve as a quick reference for implementing them. It implies a general understanding of the basics of typography, as well as HTML and CSS. All code examples shown here are supported by all modern major browsers like Chrome, Firefox, and Safari.

The Basics

Let’s get started with the general type settings: Choose your typeface with font-family, set the size of you text using font-size and the weight with font-weight. For kerning or tracking there is letter-spacing while the leading can be set with line-height. With font-style you can choose a normal or italic face and the shorthand property font allows you to combine all of the above. With color you can give your text any color, just make sure the contrast to the background is high enough to pass accessibility standards.

See the Pen Basic type settings by Jay (@jay99) on CodePen.

Loading Fonts

If you don’t want to use system fonts you can load custom fonts. To implement your own font you should convert it to woff/woff2 first, load it with @font-face and set font-display to swap, so users can see the text even if the font never loads.

  • Data privacy
  • No dependency
  • Additional configuration

See the Pen Load fonts with @font-face by Jay (@jay99) on CodePen.

Alternatively you can use Google Fonts or any other font service for that matter and use their implementation – usually with a <link> or <script>-tag in the header of your website.

  • Easy to implement
  • Character sets & font updates
  • Data privacy
  • Dependent on font service

See the Pen Load google fonts by Jay (@jay99) on CodePen.

Setting Sizes Responsively

There are countless ways of implementing font-sizes that change depending on the screen size. The following is a selection of simple and relevant ways to do so.

Adaptive Typography

This approach uses a fixed font-size that will change based on the defined media queries. It’s the classic technique, that’s accessible because users can change the font-size in their browser settings but it comes at the cost of total control over the layout.

  • Accessibility
  • Simplicity
  • Layout control
  • Can get inconsitent

See the Pen Fluid type scale with clamp by Jay (@jay99) on CodePen.

You could generate a type scale online or just write it in CSS with a combination of calc() and custom properties. It adds a layer of complexity but allows for a consistent type scale system.

  • Accessibility
  • Consistency
  • Complexity
  • Layout control

See the Pen Type scale with custom properties by Jay (@jay99) on CodePen.

Fluid Typography

For full control over that layout, the font-size can be set with relative units like vh, vw, or vmin. Now the typography scales fluidly between screen sizes but can not be easily changed by the user.

  • Layout control
  • Consistency
  • Accessibility
  • Complexity

See the Pen Fluid type scale with custom properties by Jay (@jay99) on CodePen.

Or combine that approach with clamp() to automatically set min and max values for the font sizes. No media query needed.

See the Pen Fluid type scale with custom properties by Jay (@jay99) on CodePen.

Other Responsive Type Features

The text-overlfow property can be set toellipsis to indicate that the text doesn't fit in it's parent.

Variable fonts allow for multple variantions in a font. Depending on the font, it may be able replace multiple font-files, styles or come with other features that could be useful for responsive typography like an adjustable x-height. Checkout googles list of variable fonts.
